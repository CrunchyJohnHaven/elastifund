================================================================================
RUNNING THE TESTS
================================================================================

Run all tests:
  pytest tests/ -v

Run with coverage:
  pytest tests/ --cov=src --cov-report=term-missing

Run specific test file:
  pytest tests/test_risk.py -v

Run specific test class:
  pytest tests/test_risk.py::TestRiskManager -v

Run specific test:
  pytest tests/test_risk.py::TestRiskManager::test_kill_switch_blocks_trade -v

Run with output capture disabled:
  pytest tests/ -v -s

Run with detailed local variables on failure:
  pytest tests/ -l

Generate HTML coverage report:
  pytest tests/ --cov=src --cov-report=html
  (opens htmlcov/index.html in browser)

================================================================================
DESIGN PRINCIPLES
================================================================================

1. ISOLATION
   - Each test is independent
   - Fresh database per test
   - No cross-test dependencies

2. CLARITY
   - Test names describe what they verify
   - Specific assertions (not just truthy checks)
   - Comments explain complex logic

3. COVERAGE
   - Happy paths (normal operation)
   - Error cases (limits, rejections)
   - Edge cases (insufficient data, etc.)

4. MAINTAINABILITY
   - Fixtures for common setup
   - Factory functions for complex objects
   - DRY principle (no repeated setup code)

5. SAFETY
   - Live trading blocked by default
   - Test credentials isolated
   - No real API calls in tests
