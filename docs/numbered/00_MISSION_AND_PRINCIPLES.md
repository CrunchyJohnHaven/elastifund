## Mission

Elastifund is an open, self-improving agentic operating system for real economic work.
It exists to turn research, experiments, and execution into searchable evidence so workers improve with every run.
The project has two connected missions.
The first mission is to find validated trading and non-trading loops that can create real value under explicit policy.
The second mission is to publish an honest public operating manual for how governed agent systems are built, tested, and improved.

Twenty percent of net trading profits are reserved for veteran suicide prevention.
That commitment is part of the operating mission, not a marketing add-on.

## Product Definition

Elastifund is not just a bot and it is not just a website.
It is a shared substrate for two worker families:

- Trading workers that research, simulate, rank, and optionally execute market strategies under policy.
- Non-trading workers that create economic value through business development, research, services, and customer acquisition.

Both families share system memory, evaluation, observability, workflow automation, and publishing.
The unifying principle is simple: the project does not just run agents. It improves agents.
Improvement is the product.

## What The System Optimizes For

- Evidence over promises.
- Measurable improvement over vague activity.
- Fast feedback loops over slow, ambiguous bets.
- Policy-governed autonomy over unbounded expansion.
- Reusable memory over isolated heroics.
- Honest public reporting over inflated claims.
- Narrow wedges that can be validated before generalization.

## Human And Agent Roles

John designs the constraints, infrastructure, platform choices, and risk boundaries.
JJ and future workers execute inside those boundaries.
Humans stay above the loop.
The system owns the loop within policy.
That distinction matters because the project should never imply absent oversight or undefined authority.

## Non-Negotiable Principles

### 1. Evidence First

No strategy, worker, or public claim gets promoted without evidence from tests, artifacts, or verified runtime behavior.
If the evidence is weak, the lane stays blocked.

### 2. Policy Before Autonomy

Approvals, risk caps, compliance rules, and launch gates are product features.
They are not friction to be worked around.

### 3. Shared Memory Beats Local Memory

Trading and non-trading lanes should write to the same memory and telemetry spine.
A learning system compounds only when outcomes are queryable across lanes.

### 4. Publish The Failures

Rejected strategies, missed forecasts, and failed experiments belong in the record.
Credibility comes from showing the misses as clearly as the wins.

### 5. Paper Mode By Default

The system should default to safe, inspectable operation.
Real-money deployment, risk changes, and irreversible actions require explicit escalation.

### 6. Fail Soft

Observability and external services should strengthen the system without becoming hard dependencies that break core execution.

## What Elastifund Refuses To Be

Elastifund is not a hype machine.
It is not an unbounded autonomous operator.
It is not a promise of easy profit.
It is not a place to hide uncertainty behind dashboards.
It is not a repo that mixes live, paper, and forecast results into one flattering story.
