Contributor modes

Three useful ways to enter the repo.

Observer

Read the claim surfaces before touching code.

Start with the homepage, trading board, worker board, and /live/ route so you understand what is live, forecast, and still blocked.

Runner

Boot the stack in safe mode and keep artifacts coherent.

Run the verified commands, regenerate the checked-in outputs, and leave the public story sharper than you found it.

Builder

Own one path, patch it, and verify it.

Use one owner per file, run the narrowest relevant tests, then finish with the shared hygiene and verification targets.

Trust model

Paper mode by default, explicit escalation for real-money changes.

Capital transitions, risk-limit changes, and unclear architecture choices stay behind human review.
