Verified local boot
Run from the repo root

# optional full verification

Contributor modes

Three ways to participate without pretending the complexity is zero.

Observer

Read the evidence surface.

Watch the public status board, compare the latest pipeline verdict, and understand why the launch posture is blocked before you touch code.

Runner

Boot the stack in safe mode and contribute runtime data.

Use the verified quickstart path, run paper-mode workflows, and keep the runtime artifacts green without widening any risk boundary.

Builder

Patch one lane, run the narrowest test, then the shared checks.

Codex is best used on focused implementation and test repair. Keep one owner per file, write evidence, and leave a precise handoff.

Rule

Run in paper mode by default.

Real-money transitions, risk-limit changes, and unclear architecture choices stay behind explicit human escalation.
